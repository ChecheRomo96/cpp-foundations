Lesson 00 – Configuración del entorno de trabajo (local)

Objetivos

Al finalizar esta lección, el estudiante será capaz de:

  • Instalar y verificar las herramientas necesarias para el curso
  • Clonar el repositorio y entender la estructura mínima
  • Configurar un flujo local con CMake (build fuera del árbol)
  • Compilar el proyecto en modo Debug/Release
  • Ejecutar pruebas localmente usando CTest
  • Diagnosticar errores comunes de compilación y configuración

Requisitos mínimos

Se requiere:

  • Acceso a una terminal tipo UNIX (Linux/macOS o Windows con WSL)
  • Un compilador de C++ (GCC o Clang)
  • Git
  • CMake

Recomendado:

  • Un editor como VS Code, CLion o similar
  • Un depurador (gdb o lldb)

Sistemas operativos soportados

El curso funciona en:

  • Linux (recomendado)
  • macOS
  • Windows usando WSL (recomendado sobre MSVC para este curso)

Nota: Si usas Windows, el flujo recomendado es instalar WSL2 con Ubuntu.

Instalación de herramientas

Linux (Ubuntu/Debian)

sudo apt update
sudo apt install -y build-essential git cmake

Opcional (recomendado para debug):

sudo apt install -y gdb

macOS

Instala Xcode Command Line Tools:

xcode-select --install

Instala CMake y Git con Homebrew:

brew install git cmake

Opcional (depuración):

  • lldb ya viene integrado con Xcode Command Line Tools

Windows (WSL2 + Ubuntu)

1) Instala WSL2 y Ubuntu (desde Microsoft Store). 2) Abre Ubuntu y ejecuta:

sudo apt update
sudo apt install -y build-essential git cmake

Recomendación:

  • Trabaja dentro del filesystem de WSL (ej. /home/<user>/projects)
  • Evita compilar en /mnt/c/ para no degradar el rendimiento

Verificación de instalación

Ejecuta estos comandos y confirma que muestran versiones:

g++ --version
git --version
cmake --version

Si alguno falla:

  • No continúes con el curso todavía
  • Resuelve la instalación primero

Flujo de compilación local con CMake

Este curso usa un flujo estándar:

  • Configurar el proyecto en una carpeta de build
  • Compilar desde esa carpeta
  • Ejecutar pruebas

Build fuera del árbol (recomendado)

Desde la raíz del repo:

mkdir -p build
cmake -S . -B build
cmake --build build

Explicación:

  • cmake -S . -B build : configura el proyecto (source = ., build = build)
  • cmake –build build : compila

Configuración Debug y Release

CMake soporta modos de compilación. En generadores tipo Makefiles/Ninja se define con CMAKE_BUILD_TYPE:

Debug:

cmake -S . -B build-debug -DCMAKE_BUILD_TYPE=Debug
cmake --build build-debug

Release:

cmake -S . -B build-release -DCMAKE_BUILD_TYPE=Release
cmake --build build-release

Recomendación del curso:

  • Debug para desarrollo y depuración
  • Release para medir rendimiento (cuando aplique)

Limpiar build (cuando sea necesario)

Si cambias algo importante del entorno o el CMakeLists, a veces conviene borrar build:

rm -rf build
mkdir -p build
cmake -S . -B build
cmake --build build

Ejecutar pruebas localmente

Este repositorio está diseñado para correr pruebas localmente si están configuradas. El estándar de CMake para pruebas es CTest.

Asegurar que el proyecto está compilado

cmake -S . -B build
cmake --build build

Ejecutar pruebas con CTest

Ejecutar todas las pruebas:

ctest --test-dir build

Ejecutar con salida detallada (recomendado cuando falle algo):

ctest --test-dir build --output-on-failure

Ejecutar pruebas en paralelo (si tu máquina lo soporta):

ctest --test-dir build -j 4 --output-on-failure

Nota:

  • Si "ctest" dice "No tests were found", entonces el proyecto todavía no tiene pruebas registradas o la opción de pruebas no está habilitada.

Ejecutar ejecutables localmente

Dependiendo de cómo esté definido el proyecto, los ejecutables suelen quedar en:

  • build/ (o subcarpetas internas)

Ejemplo genérico:

./build/<nombre_del_ejecutable>

Si no sabes el nombre del ejecutable:

ls build

Si el proyecto usa subdirectorios:

find build -maxdepth 3 -type f -perm -111

Depuración local (mínimo indispensable)

Linux/WSL (gdb)

Compila en Debug y ejecuta:

gdb ./build-debug/<ejecutable>

Comandos útiles dentro de gdb:

  • run
  • bt
  • break <archivo:linea>
  • next
  • step
  • print <variable>

macOS (lldb)

Compila en Debug y ejecuta:

lldb ./build-debug/<ejecutable>

Comandos útiles dentro de lldb:

  • run
  • bt
  • breakpoint set –file <archivo> –line <n>
  • next
  • step
  • frame variable

Errores comunes y soluciones rápidas

CMake no encuentra compilador

Síntoma:

  • CMake dice que no encuentra C/C++ compiler

Solución:

  • Instala build-essential (Linux/WSL)
  • En macOS instala xcode-select –install
  • Revisa que g++ –version funcione

CMake se queda "atorado" con config vieja

Síntoma:

  • Cambiaste cosas y no se reflejan

Solución:

  • Borra build y regenera
rm -rf build
cmake -S . -B build
cmake --build build

No aparecen pruebas

Síntoma:

  • ctest: No tests were found

Posibles causas:

  • Aún no hay pruebas registradas
  • El proyecto requiere una opción para habilitar pruebas

Acción:

  • Revisa el README del repo o la lección correspondiente a pruebas
