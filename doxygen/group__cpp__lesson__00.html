Lesson 00 – Configuración del entorno de trabajo (local)

Objetivos

Al finalizar esta lección, el estudiante será capaz de:

  • Crear su repositorio personal usando un template de GitHub
  • Clonar y preparar el proyecto localmente
  • Instalar y verificar las herramientas necesarias
  • Compilar el proyecto usando CMake
  • Ejecutar pruebas localmente con CTest
  • Entender las reglas de protección del curso
  • Realizar su commit inicial de forma correcta

Requisitos mínimos

Se requiere:

  • Acceso a una terminal tipo UNIX (Linux/macOS o Windows con WSL)
  • Un compilador de C++ (GCC o Clang)
  • Git
  • CMake

Recomendado:

  • Un editor de código (VS Code, CLion, etc.)
  • Un depurador (gdb o lldb)

Sistemas operativos soportados

El curso funciona en:

  • Linux (recomendado)
  • macOS
  • Windows usando WSL2

Instalación de herramientas

Linux (Ubuntu/Debian)

sudo apt update
sudo apt install -y build-essential git cmake gdb

macOS

xcode-select --install
brew install git cmake

Windows (WSL2 + Ubuntu)

sudo apt update
sudo apt install -y build-essential git cmake gdb

Verificación de instalación

g++ --version
git --version
cmake --version

Si alguno de estos comandos falla, el entorno no está listo.

Uso del repositorio como template

El repositorio cpp-foundations se utiliza como template.

Este paso se realiza una sola vez, directamente en GitHub.

Pasos:

  1. Ir al repositorio base: https://github.com/ChecheRomo96/cpp-foundations
  2. Hacer clic en Use this template
  3. Crear el repositorio personal del curso

El repositorio resultante es independiente y pertenece al estudiante.

Clonado del repositorio del estudiante

Una vez creado el repositorio a partir del template, se clona localmente:

git clone https://github.com/<tu-usuario>/cpp-foundations.git
cd cpp-foundations
git status

A partir de este punto, todo el trabajo se realiza sobre este repositorio.

Protección del curso (COURSE_GUARD_ENABLED)

El repositorio incluye un mecanismo de protección del curso.

La variable:

COURSE_GUARD_ENABLED = true

indica que el repositorio está operando en modo curso.

Cuando está habilitada:

  • El CI valida que solo se modifique la carpeta student/
  • Cambios fuera de student/ provocan falla automática
  • La estructura del curso queda protegida

Esta variable no debe ser modificada por el estudiante.

Estructura mínima del repositorio

cpp-foundations/
├── student/
│ ├── include/
│ └── src/
├── docs/
│ └── topics/
├── CMakeLists.txt
├── .gitignore
└── README.md

Regla del curso:

  • El estudiante solo modifica student/

Regla de archivos ignorados (.gitignore)

Los archivos generados por la compilación no deben versionarse.

En particular:

build/

debe estar listado en .gitignore.

Ejemplo mínimo:

build/
build-*/
*.o
*.out
*.exe
CMakeCache.txt
CMakeFiles/

Subir archivos de build invalida el commit.

Compilación local con CMake

Flujo recomendado (build fuera del árbol):

mkdir -p build
cmake -S . -B build
cmake --build build

Ejecución de pruebas locales

ctest --test-dir build --output-on-failure

Commit inicial obligatorio

Una vez verificado que el proyecto:

  • Compila correctamente
  • Ejecuta pruebas sin errores

el estudiante debe realizar su commit inicial.

Pasos del commit inicial

git status
git add student/
git commit -m "Lesson 00: environment setup and local build verified"
git push

Si aún no existen archivos en student/, crea uno mínimo:

touch student/src/.init
git add student/src/.init
git commit -m "Lesson 00: initial student workspace"
git push

Checklist final

Antes de pasar a Lesson 01, el estudiante debe poder ejecutar:

git clone https://github.com/<tu-usuario>/cpp-foundations.git
cd cpp-foundations
cmake -S . -B build
cmake --build build
ctest --test-dir build --output-on-failure
git status
git add student/
git commit -m "Lesson 00: environment setup and local build verified"
git push

Si todos estos pasos funcionan correctamente:

  • El entorno está listo
  • Git está correctamente configurado
  • El estudiante puede avanzar a la siguiente lección
