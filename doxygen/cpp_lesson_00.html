Lesson 00 – Introducción al repositorio y flujo de trabajo

Introducción al repositorio cpp-foundations, su estructura y el flujo de trabajo esperado para el curso.

Objetivos

Al finalizar esta lección, el estudiante será capaz de:

  • Comprender el propósito del repositorio cpp-foundations
  • Navegar correctamente la estructura del proyecto
  • Entender cómo se organiza el contenido del curso
  • Seguir el flujo básico de trabajo con Git y GitHub
  • Saber dónde escribir código y dónde no

¿Qué es cpp-foundations?

cpp-foundations es un repositorio educativo diseñado para:

  • Enseñar C++ desde fundamentos reales
  • Promover buenas prácticas desde el inicio
  • Simular un entorno de trabajo profesional
  • Separar claramente código del alumno y código base

Este repositorio NO es solo una colección de ejemplos, es un entorno de aprendizaje estructurado.

Estructura del repositorio

La estructura general del repositorio es la siguiente:

cpp-foundations/
├── student/
│ ├── include/
│ └── src/
├── docs/
│ └── topics/
├── README.md
└── CMakeLists.txt

Carpeta student/

Esta es la ÚNICA carpeta que el estudiante debe modificar.

student/
├── include/ // Headers creados por el alumno
└── src/ // Código fuente del alumno

Reglas importantes:

  • Todo el código del alumno vive aquí
  • No modificar archivos fuera de student/
  • El CI del repositorio valida esta regla

Carpeta docs/

Contiene la documentación del curso.

docs/
└── topics/
├── Lesson00.dox
├── Lesson01.dox
└── ...

Estas lecciones se generan automáticamente usando Doxygen.

Flujo de trabajo esperado

El flujo de trabajo básico es:

  1. Clonar el repositorio
  2. Crear o modificar archivos SOLO en student/
  3. Hacer commits claros y pequeños
  4. Subir cambios a GitHub
  5. Verificar que el CI pase correctamente

Git básico (mínimo indispensable)

Clonar el repositorio:

git clone <url-del-repositorio>

Ver el estado del repositorio:

git status

Agregar cambios:

git add student/

Crear un commit:

git commit -m "Lesson 01: variables and scope"

Subir cambios:

git push

Reglas de commits

Buenas prácticas:

  • Commits pequeños
  • Mensajes claros
  • Un commit por avance lógico

Ejemplos correctos:

  • "Lesson 00: project setup"
  • "Lesson 01: data types examples"

Ejemplos incorrectos:

  • "changes"
  • "final"
  • "fix"

Continuous Integration (CI)

El repositorio utiliza GitHub Actions para:

  • Verificar que solo se modifique student/
  • Compilar el código
  • Generar reportes

Si el CI falla:

  • El código NO se considera válido
  • El cambio debe corregirse

Aprender a leer errores del CI es parte del curso.

Expectativas del curso

Se espera que el estudiante:

  • Lea las lecciones antes de programar
  • Entienda el código que escribe
  • No copie código sin comprenderlo
  • Use el repositorio como herramienta de aprendizaje

Este curso prioriza:

  • Claridad sobre velocidad
  • Fundamentos sobre trucos
  • Código correcto sobre código rápido

Idea clave

El repositorio es parte del curso. Aprender a trabajar con estructura, control de versiones y reglas es tan importante como aprender C++.
