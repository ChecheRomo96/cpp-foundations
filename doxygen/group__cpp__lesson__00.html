Lesson 00 – Configuración del entorno de trabajo (local)

Objetivos

Al finalizar esta lección, el estudiante será capaz de:

  • Instalar y verificar las herramientas necesarias para el curso
  • Clonar el repositorio y entender la estructura mínima
  • Configurar un flujo local con CMake (build fuera del árbol)
  • Compilar el proyecto en modo Debug/Release
  • Ejecutar pruebas localmente usando CTest
  • Realizar su commit inicial correctamente
  • Entender las reglas de protección del repositorio

Requisitos mínimos

Se requiere:

  • Acceso a una terminal tipo UNIX (Linux/macOS o Windows con WSL)
  • Un compilador de C++ (GCC o Clang)
  • Git
  • CMake

Recomendado:

  • Un editor como VS Code, CLion o similar
  • Un depurador (gdb o lldb)

Sistemas operativos soportados

El curso funciona en:

  • Linux (recomendado)
  • macOS
  • Windows usando WSL (recomendado sobre MSVC para este curso)

Instalación de herramientas

Linux (Ubuntu/Debian)

sudo apt update
sudo apt install -y build-essential git cmake gdb

macOS

xcode-select --install
brew install git cmake

Windows (WSL2 + Ubuntu)

sudo apt update
sudo apt install -y build-essential git cmake gdb

Verificación de instalación

g++ --version
git --version
cmake --version

Clonar y preparar el repositorio

git clone https://github.com/ChecheRomo96/cpp-foundations.git
cd cpp-foundations
git status

Protección del curso (COURSE_GUARD_ENABLED)

Este repositorio incluye un mecanismo de protección del curso.

La variable:

COURSE_GUARD_ENABLED = true

indica que el repositorio está operando en modo curso.

Cuando esta variable está habilitada:

  • El CI valida que solo se modifique la carpeta student/
  • Cambios fuera de student/ provocan falla automática
  • El flujo de trabajo del curso se hace cumplir estrictamente

Esta variable NO debe ser modificada por el estudiante.

Su propósito es:

  • Proteger la estructura del curso
  • Evitar modificaciones accidentales
  • Garantizar evaluaciones justas y consistentes

Estructura mínima del repositorio

cpp-foundations/
├── student/
│ ├── include/
│ └── src/
├── docs/
│ └── topics/
├── CMakeLists.txt
├── .gitignore
└── README.md

Regla del curso:

  • El estudiante solo modifica student/

Regla de archivos ignorados (.gitignore)

Los archivos generados por la compilación no deben versionarse.

En particular, la carpeta:

build/

debe estar listada en .gitignore.

Ejemplo mínimo de .gitignore usado en este curso:

build/
build-*/
*.o
*.exe
*.out
CMakeCache.txt
CMakeFiles/

Subir archivos de build/ invalida el commit.

Flujo de compilación local con CMake

mkdir -p build
cmake -S . -B build
cmake --build build

Ejecutar pruebas localmente

ctest --test-dir build --output-on-failure

Commit inicial obligatorio

Una vez que el proyecto:

  • Compila correctamente
  • Ejecuta pruebas sin errores

el estudiante debe realizar su commit inicial.

Pasos del commit inicial

git status
git add student/
git commit -m "Lesson 00: environment setup and local build verified"
git push

Si aún no existen archivos en student/, crea uno mínimo:

touch student/src/.init
git add student/src/.init
git commit -m "Lesson 00: initial student workspace"
git push

Checklist final

Antes de pasar a Lesson 01, el estudiante debe poder ejecutar:

git clone https://github.com/ChecheRomo96/cpp-foundations.git
cd cpp-foundations
cmake -S . -B build
cmake --build build
ctest --test-dir build --output-on-failure
git status
git add student/
git commit -m "Lesson 00: environment setup and local build verified"
git push

Si todos estos pasos funcionan correctamente:

  • El entorno está listo
  • Git está correctamente configurado
  • El estudiante puede avanzar a la siguiente lección
