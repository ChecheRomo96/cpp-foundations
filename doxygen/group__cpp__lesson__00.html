Lesson 00.1 – Instalación y verificación del entorno

Objetivos

  • Instalar herramientas necesarias
  • Verificar versiones
  • Identificar un entorno compatible

Requisitos mínimos

Se requiere:

  • Terminal tipo UNIX (Linux/macOS o Windows con WSL2)
  • Compilador C++ (GCC o Clang)
  • Git
  • CMake

Sistemas operativos soportados

  • Linux (recomendado)
  • macOS
  • Windows con WSL2 (recomendado)

Instalación de herramientas

Linux (Ubuntu/Debian)

sudo apt update
sudo apt install -y build-essential git cmake gdb

macOS

xcode-select --install
brew install git cmake

Windows (WSL2 + Ubuntu)

sudo apt update
sudo apt install -y build-essential git cmake gdb

Nota:

  • Trabaja dentro de /home/<usuario>/ para mejor rendimiento.

Verificación del entorno

g++ --version
git --version
cmake --version

Si alguno falla, el entorno NO está listo.

Lesson 00.2 – Workflow del curso: Fork + Upstream

Modelo del curso

  • Repositorio oficial (upstream): https://github.com/ChecheRomo96/cpp-foundations
  • Repositorio del estudiante (origin): fork del upstream

Crear el fork (GitHub)

1) Entra al repositorio oficial del curso 2) Haz clic en Fork 3) Crea el fork en tu cuenta

Clonar tu fork

git clone https://github.com/<tu-usuario>/cpp-foundations.git
cd cpp-foundations

Configurar upstream

git remote add upstream https://github.com/ChecheRomo96/cpp-foundations.git
git remote -v

Actualizar desde terminal

Recomendación: usar –no-edit para evitar el editor en el merge.

git fetch upstream
git merge upstream/main --no-edit

Actualizar desde GitHub (Sync fork)

1) Entra a tu fork en GitHub 2) Haz clic en Sync fork 3) Selecciona Update branch Después actualiza local:

git pull origin main

Protección del curso

En CI, el guard está activado por defecto:

  • En forks: siempre ON
  • En upstream: puede override con repo variable

Lesson 00.3 – Compilar y correr pruebas localmente

Compilación local con CMake

mkdir -p build
cmake -S . -B build
cmake --build build

Ejecutar pruebas locales

ctest --test-dir build --output-on-failure

Commit inicial obligatorio

git status
git add student/
git commit -m "Lesson 00: environment setup and local build verified"
git push

Si aún no existen archivos en student/, crea uno mínimo:

touch student/src/.init
git add student/src/.init
git commit -m "Lesson 00: initial student workspace"
git push

Checklist final

git clone https://github.com/<tu-usuario>/cpp-foundations.git
cd cpp-foundations
git remote add upstream https://github.com/ChecheRomo96/cpp-foundations.git
cmake -S . -B build
cmake --build build
ctest --test-dir build --output-on-failure
git add student/
git commit -m "Lesson 00: environment setup and local build verified"
git push

Lesson 00.4 – .gitignore y archivos generados

Qué es .gitignore

El archivo .gitignore indica a Git qué archivos y carpetas deben ser ignorados (no rastreados ni versionados), aunque existan en tu máquina.

Archivos de compilación

En este curso, los archivos generados por compilación deben permanecer locales. En particular:

build/

nunca debe subirse al repositorio.

Ejemplo recomendado

# --------------------------------------------------
# Build directories
# --------------------------------------------------
build/
cmake-build-*/
# --------------------------------------------------
# CMake generated files
# --------------------------------------------------
CMakeCache.txt
CMakeFiles/
cmake_install.cmake
CTestTestfile.cmake
Makefile
# --------------------------------------------------
# Coverage files (local only)
# --------------------------------------------------
*.gcda
*.gcno
*.gcov
coverage.info
coverage.filtered.info
coverage_html/
# --------------------------------------------------
# IDE / Editor files
# --------------------------------------------------
.vscode/
.idea/
*.swp
*.swo
# --------------------------------------------------
# Operating system files
# --------------------------------------------------
.DS_Store
Thumbs.db
# --------------------------------------------------
# Doxygen output (generated)
# --------------------------------------------------
docs_out/

Regla del curso

Subir archivos generados (por ejemplo build/) invalida el commit.

Lesson 00.5 – Troubleshooting y fixes comunes

Merge incompleto (MERGE_HEAD exists)

Si Git indica que hay un merge sin concluir:

Terminar el merge (si no hay conflictos):

git commit --no-edit

Cancelar el merge (si quieres reiniciar):

git merge --abort

Arreglar un CI fallido por archivos fuera de student/

git restore --staged .
git restore .
git add student/
git commit -m "Fix: only student changes"
git push

Regla de oro

SIEMPRE:

git add student/

NUNCA:

git add .
