Lesson 00 – Introducción al repositorio y flujo de trabajo

Objetivos

Al finalizar esta lección, el estudiante será capaz de:

  • Comprender el propósito del repositorio cpp-foundations
  • Navegar correctamente la estructura del proyecto
  • Entender cómo se organiza el contenido del curso
  • Seguir el flujo básico de trabajo con Git y GitHub
  • Saber dónde escribir código y dónde no

¿Qué es cpp-foundations?

cpp-foundations es un repositorio educativo diseñado para:

  • Enseñar C++ desde fundamentos reales
  • Promover buenas prácticas desde el inicio
  • Simular un entorno de trabajo profesional
  • Separar claramente código del alumno y código base

Este repositorio NO es solo una colección de ejemplos, es un entorno de aprendizaje estructurado.

Estructura del repositorio

cpp-foundations/
├── student/
│ ├── include/
│ └── src/
├── docs/
│ └── topics/
├── README.md
└── CMakeLists.txt

Carpeta student/

Esta es la ÚNICA carpeta que el estudiante debe modificar.

student/
├── include/
└── src/

Reglas importantes:

  • Todo el código del alumno vive aquí
  • No modificar archivos fuera de student/
  • El CI del repositorio valida esta regla

Flujo de trabajo esperado

  1. Clonar el repositorio
  2. Crear o modificar archivos SOLO en student/
  3. Hacer commits claros y pequeños
  4. Subir cambios a GitHub
  5. Verificar que el CI pase correctamente

Git básico (mínimo indispensable)

git clone <url-del-repositorio>
git status
git add student/
git commit -m "Lesson 00: repo workflow"
git push

Continuous Integration (CI)

El repositorio utiliza GitHub Actions para verificar:

  • Que solo se modifique student/
  • Que el proyecto compile
  • Que los reportes se generen correctamente

Si el CI falla, el cambio no se considera válido.
