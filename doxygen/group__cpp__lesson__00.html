Lesson 00 – Introducción al repositorio y flujo de trabajo

Onboarding técnico al curso cpp-foundations: entorno, herramientas, estructura del repositorio y flujo de trabajo.

Objetivos

Al finalizar esta lección, el estudiante será capaz de:

  • Comprender el propósito del repositorio cpp-foundations
  • Navegar correctamente la estructura del proyecto
  • Entender cómo se organiza el contenido del curso
  • Seguir el flujo básico de trabajo con Git y GitHub
  • Saber dónde escribir código y dónde no

¿Qué es cpp-foundations?

cpp-foundations es un repositorio educativo diseñado para:

  • Enseñar C++ desde fundamentos reales
  • Promover buenas prácticas desde el inicio
  • Simular un entorno de trabajo profesional
  • Separar claramente código del alumno y código base

Este repositorio NO es solo una colección de ejemplos, es un entorno de aprendizaje estructurado.

Estructura del repositorio

cpp-foundations/
├── student/
│ ├── include/
│ └── src/
├── docs/
│ └── topics/
├── README.md
└── CMakeLists.txt

Carpeta student/

Esta es la ÚNICA carpeta que el estudiante debe modificar.

student/
├── include/
└── src/

Reglas importantes:

  • Todo el código del alumno vive aquí
  • No modificar archivos fuera de student/
  • El CI del repositorio valida esta regla

Flujo de trabajo esperado

  1. Clonar el repositorio
  2. Crear o modificar archivos SOLO en student/
  3. Hacer commits claros y pequeños
  4. Subir cambios a GitHub
  5. Verificar que el CI pase correctamente

Git básico (mínimo indispensable)

git clone <url-del-repositorio>
git status
git add student/
git commit -m "Lesson 00: repo workflow"
git push

Continuous Integration (CI)

El repositorio utiliza GitHub Actions para verificar:

  • Que solo se modifique student/
  • Que el proyecto compile
  • Que los reportes se generen correctamente

Si el CI falla, el cambio no se considera válido.

Configuración del entorno de trabajo

Antes de escribir código, es necesario preparar el entorno.

Sistemas operativos soportados

El curso está diseñado para funcionar en:

  • Linux (recomendado)
  • macOS
  • Windows (usando WSL)

Se espera que el estudiante tenga acceso a una terminal tipo UNIX.

Instalación de herramientas necesarias

Las herramientas mínimas requeridas son:

  • Un compilador de C++ (g++ o clang++)
  • Git
  • CMake
  • Un editor de código

Linux

sudo apt update
sudo apt install build-essential git cmake

macOS

Instalar Xcode Command Line Tools:

xcode-select --install

Instalar Homebrew (si no está instalado):

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Instalar herramientas:

brew install git cmake

Windows (WSL)

Se recomienda usar WSL con Ubuntu.

Una vez dentro de WSL:

sudo apt update
sudo apt install build-essential git cmake

Verificación de herramientas

Verifica que las herramientas estén instaladas correctamente:

g++ --version
git --version
cmake --version

Si alguno de estos comandos falla, el entorno NO está listo.

Uso básico de la terminal

La terminal es una herramienta fundamental para este curso.

Navegación básica

pwd // muestra el directorio actual
ls // lista archivos
cd folder // cambia de directorio
cd .. // sube un nivel

Manejo de archivos

mkdir test
touch main.cpp
rm archivo.cpp

Git básico (mínimo indispensable)

Clonar el repositorio:

git clone <url-del-repositorio>
cd cpp-foundations

Ver el estado del repositorio:

git status

Agregar cambios:

git add student/

Crear un commit:

git commit -m "Lesson 00: environment setup"

Subir cambios:

git push

Flujo de trabajo esperado

El flujo de trabajo estándar del curso es:

  1. Leer la lección
  2. Implementar código en student/
  3. Probar localmente
  4. Crear commits pequeños y claros
  5. Subir cambios
  6. Verificar que el CI pase

Continuous Integration (CI)

El repositorio utiliza GitHub Actions para verificar:

  • Que solo se modifique student/
  • Que el proyecto compile
  • Que los reportes se generen correctamente

Si el CI falla, el cambio NO se considera válido.

Aprender a leer errores del CI es parte del curso.
